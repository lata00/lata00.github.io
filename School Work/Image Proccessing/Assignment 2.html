3.1      Grayscale Conversion

First, I will want to convert the image to grayscale, since most of the image processing techniques are designed to work on grayscale or even binary images. However, I need to be careful about how I do this The paper in the center of the image is, in essence, a white rectangle, while the surrounding background is generally some darker or different colored region. So, I will want to preserve, as best as I can, the color information that the image contains. During the course of this assignment, I noticed that when using the MATLAB built in grayscale converter led to poor results when the background was yellow. When I looked into why that might be, I came across the following definition of the MATLAB grayscale conversion:

rgb2gray converts RGB values to grayscale values by forming a weighted sum of the R, G, and B components:

0.2989 * R + 0.5870 * G + 0.1140 * B

Which is to say, the resulting grayscale image is not a true average of the color, but contains some weighting that is meant to correspond to human perception of the colors. While this is likely useful for making more natural grayscale images it does not do us any favors since it leads to something like yellow, which is just an absence of blue, appearing very similar to white. To make sure that all colors are perceived equally by the algorithm, Ill use the following function to convert my colors to grayscale:

g = uint8(ceil(sum(im,3)/3))

 

With this function, all colors area weighted equally, so a picture with a yellow background will have more contrast between the foreground and the background. This can clearly be seen with example image 18:

3.2      Thresholding

In order to best detect the edge of the paper, it can also be helpful to convert the grayscale image to a binary image. This is typically done by a method known as thresholding, where any pixels above a certain threshold value is converted to white, and anything below that becomes black. Ideally, this threshold value will be such that only the paper turns white, and the background becomes black. Looking at the histogram of an image can give us an idea of where this should be. For the grayscale version of the yellow background image above, this looks like:

You can see we have two distinct peaks in this image, one representing the papers pixels and one representing the backgrounds. The most direct way to determine the appropriate threshold value is to find the average pixel value. This gives us the average brightness level of the image, and presumably the paper pixels will fall somewhere above this brightness level and the background will fall somewhere below this brightness level. 4.1      Region Detection

To detect various regions of the pre-filtered image, one can use the MATLAB function regionprops(). This will detect portions of the image that are connected and provide properties about the connected areas, such as area, centroid, etc. One particular property that is interesting for our purposes is the BoundingBox property, which specifies the rectangle that encompasses the connected regions detected in the image. Assuming we have properly thresholded and connected the regions, then there should only be one region in our image that represents the area of the paper, and this will give us the exact rectangular region that we are interested in. I can use the ratio of rectangle area to white space as a metric of the validity of this method and loop over the shape size threshold until I achieve a reasonable value, like so:

for i = 0:10000:(h*w)

bw1 = bwareaopen(f,i);

bw2 = bwconvhull(bw1);

W = sum(sum(bw2));

end

end

5       Algorithm

My final algorithm is a combination of the region detection and the Area Detection. If the region search appears to fail by identifying too large a region, it falls back on the area search to attempt to identify the paper:

function [sx, sy, sWidth, sHeight] = auto_crop ( im )

h = size(im,1);

w = size(im,2);

g = uint8(ceil(sum(im,3)/3));

gAvg = double(mean(mean(g)));

gMax = double(max(max(g)));

T = max([0 gAvg/gMax]);

f = imbinarize(g,T);

ColAvg = mean(f,1);

RowAvg = mean(f,2);

fAvg = mean(mean(f));

for i = 0:10000:(h*w)

bw1 = bwareaopen(f,i);

bw2 = bwconvhull(bw1);

W = sum(sum(bw2));

RP = regionprops(bw2);

BB = RP.BoundingBox;

A = BB(3)*BB(4);

if A/W < 1.15

break

end

end

rect1 = BB;

if BB(3)*BB(4)/(h*w) < .8

sx = BB(1);

sy = BB(2);

sWidth = BB(3);

sHeight = BB(4);

else

for i = 1:h

if RowAvg(i)>fAvg

y1 = i;

break

end

end

for i = h:-1:1

if RowAvg(i)>fAvg

y2 = i;

break

end

end

for i = 1:w

if ColAvg(i)>fAvg

x1 = i;

break

end

end

for i = w:-1:1

if ColAvg(i)>fAvg

x2 = i;

break

end

end

W = x2 - x1;

H = y2 - y1;

sx = x1;

sy = y1;

sWidth = W;

sHeight = H;

end
