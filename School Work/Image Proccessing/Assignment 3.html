Introduction

For this assignment, I would like to write an algorithm that can detect the lane lines. For example, I would like to take a picture like the following:

And automatically detect the lane lines of the lane that the car is in, as illustrated here:

The algorithm will return the parameters ρ and θ for each lane line, where ρ is the distance to the origin of the point along the line closest to the origin, and θ is the angle of the line relative to the x-axis. These parameters are commonly referred to as the Hough Space representation of a line, which strongly suggests that our solution should utilize a Hough Transform, a common technique for detecting lines in an image.

 

Hough Transform

The Hough transform, as Ive alluded to, is a transformation that can be used to detect lines in an image. It achieves this by considering all possible lines that pass through a point and narrowing to the most prominent lines through a voting system. Before we can get to this point, though, we must understand how a line is represented. The conventional notation for a line, y = mx + b, becomes unwieldy for this purpose, since the possible values of both m and b are unbounded, leading to potentially very large values or undefined lines. In order to get around this, Hough space is utilized, where a line is represented as:

The transform can be visualized better using the following illustration:

Figure 1: Cartesian Space vs. Hough Space

This means that each point in cartesian space will transform to a sinusoidal curve in Hough space, and co-linear points will correspond to intersections of those sinusoids. This gives us a plot that looks something like:

Figure 2: Example Hough Transform

You can see there are what appear to be many different sinusoidal curves on top of one another, with regions of more and less intersection. The regions with the greatest number of intersections will correspond to the strongest lines in the image. I will utilize this property as a basis for my lane detection algorithm, along with pre- and post-filtering of the images.

Algorithm

The algorithm I developed for this project can be found here. I will explain each step and its purpose here.

ROI Mask

I begin by establishing a region of interest mask that will attempt to filter extraneous parts of the image out from the final processing. The mask I ultimately settled on has the following form:

Figure 3: ROI mask used for this exercise

This was done by hand, and may not be a perfect match to the real region the road exists in, but it achieves much cleaner results than if the line detection was performed without the mask. Ill also note that this isnt applied until after the edge detection, to avoid the prominent edges that would occur were it to be applied before this.

Grayscale Conversion, Contrast Adjust, and Edge Detection

In order to perform edge detection on the image, which is necessary for the Hough transform to work properly, I use my own conversion method. The MatLab standard grayscale conversion uses a weighting between the three channels, but I prefer to use a pure average to ensure that colors like yellow dont get a reduced contrast. I also perform an automated contrast correction with MATLABs imadjust() function.

Next, I perform the edge detection. For this I use the built-in canny edge detection function with a threshold of .2 and a standard deviation of .5. These were empirically determined to provide the cleanest detection of the edges I am hoping to discern with minimal noise. These four stages of processing are illustrated here:

Figure 4: Stages of image pre-processing

Hough Transform & Mask

The Hough Transform is performed using MATLABs standard hough() function. Ive adjusted the rho and theta resolutions to .5. I also generate a mask that I apply to the Hough transform that filters any regions corresponding to angles that are not likely to correspond to the lanes I am interested in.

 

Hough Peaks & Threshold

I use MATLABs houghpeaks() function to identify the peaks of the Hough transform performed in the previous step, which will correspond to the most prominent lines in the image. Ive placed this step in a while loop, since it is sometimes the case, using the default thresholding, that one or both of the lane lines are not identified. The while loop allows for the algorithm to dynamically decrease the threshold value until it is able to identify at least one line for each lane line.

Figure 5: Masked Hough Transform

Obtaining Lines

To convert the Hough peak information into line information, I make use of the houghlines() function, which takes as input my Hough transform and the peaks I identified and converts them into meaningful values, i.e. rho and theta. It returns these values in a struct, so I must extract the information into a matrix. Once I have the information formatted properly, I can split the lines corresponding to each lane into separate arrays and average the values of the lines identified.

Figure 6: Lines identified by Hough Transform. These are averaged to obtain final result.

Results & Conclusions

Applying this algorithm to the provided data achieves roughly 90% accuracy by the metrics defined for this test. Many of the frame that it fails on are regions where lane lines are temporarily out of view or where the lanes have suddenly widened or are turning. There also appear to be some frames where other features are interfering with the line identification. These things could be resolved by adding a more clearly defined range of angles that are acceptable as lanes, or even adding default lane assumptions that the algorithm can fall back on when obvious lane lines are absent. These options offer the possibility of increasing my hit count several more percentage points, I would guess, but they have the drawback of making our system less flexible and more attached to our assumptions of what the lane lines should be rather than what the car is actually seeing.

detect_lanes.m

function [rho1, theta1, rho2, theta2] = detect_lanes(im)

close all

%Get size of image

width = size(im,2);

height = size(im,1);

%Define ROI mask that roughly captures lane we are interested in

mask = roipoly(im,[300 350 500 185],[185 185 340 340]);

%Convert color image to grayscale, adjust contrast, and apply canny edge filter

g = uint8(ceil(sum(im,3)/3));

f = imadjust(g);

E = edge(f,'Canny',.2,.5);

%Apply ROI mask

E = E.*mask;

%Execute Hough Trasnform

[H,T,R] = hough(E,'RhoResolution',0.5,'Theta',-90:0.5:89);

%Define and apply mask of hough trasnform to eliminate angles that are not

%expected to occur

hmask = ones(size(H,1), size(H,2));

hmask(:,1:75)=0;

hmask(:,125:200)=0;

hmask(:,300:359)=0;

H = hmask.*H;

%Define peak threshold and establish positive and negative line variables

thresh = .5;

plines = [];

nlines = [];

%Loop to find the peaks of the hough transformation, decreasing sensitivity

%until both positive and negative lines are found

while isempty(plines) || isempty(nlines)

if thresh < 0

break

end

P = houghpeaks(H,10,'Threshold',thresh*max(H(:)));

a = find(or(P(:,2)==1,P(:,2)==359));

P(a,:) = [];

lines = houghlines(g,T,R,P,'FillGap',5,'MinLength',7);

C = struct2cell(lines);

thetadeg = cell2mat(C(3,:));

rho = cell2mat(C(4,:));

for i = 1:size(thetadeg,2)

if thetadeg(i) > 0

thetadeg(i) = thetadeg(i) - 180;

rho(i) = -rho(i);

end

end

theta = deg2rad(thetadeg);

lines = [rho;theta]';

plines = find(lines(:,1)>0);

nlines = find(lines(:,1)<0);

thresh = thresh - .01;

end

for i = 1:size(plines,2)

pline = mean(lines(plines,:),1);

end

for i = 1:size(nlines,2)

nline = mean(lines(nlines,:),1);

end

rho1 = nline(1);

theta1 = nline(2);

rho2 = pline(1);

theta2 = pline(2);

end
